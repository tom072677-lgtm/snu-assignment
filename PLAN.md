# PLAN: 모바일 지도 빈 화면 수정

## 목표
모바일에서 지도 탭이 빈 화면으로 보이고 위치 권한 요청도 뜨지 않는 문제를 해결한다.

## 가정
- 지도 탭 클릭 시 카카오 SDK 초기화가 실패하면 이후 위치 권한 요청까지 실행되지 않는다.
- 위치 권한은 지도 자체 표시와 분리되어야 한다. 위치 권한이 없어도 캠퍼스 지도/마커 또는 안내 메시지는 보여야 한다.
- HTTPS가 아니거나 권한이 거부된 환경에서는 권한 요청이 뜨지 않을 수 있으므로 사용자에게 상태를 표시해야 한다.

## 접근
1. 카카오 SDK 준비 여부를 확인하고 실패 시 빈 화면 대신 안내 메시지를 표시한다.
   - verify: `kakao`가 없어도 JS 예외로 멈추지 않는다.
2. 지도 초기화와 위치 권한 요청을 분리한다.
   - verify: 지도 초기화 실패/위치 권한 부재 각각의 메시지가 따로 나온다.
3. 지도 탭 진입 시 캠퍼스 주요 위치 마커를 표시한다.
   - verify: 위치 권한을 허용하지 않아도 지도 탭에 볼 내용이 있다.
4. 모바일에서 숨김 탭 표시 직후 지도 레이아웃을 재계산한다.
   - verify: `relayout()` 호출 경로가 있다.
5. 문법 검증 후 커밋/푸시한다.
   - verify: `node --check script.js` 통과.
