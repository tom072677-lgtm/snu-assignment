# PLAN: 지도 현재위치 방향 화살표 안정화

## 목표
휴대폰이 가만히 있을 때도 방향 화살표는 표시하되, 센서 노이즈로 화살표가 요동치는 현상을 줄인다.

## 가정
- 현재 요동은 `deviceorientation`/`webkitCompassHeading` 값을 매 이벤트마다 즉시 SVG 회전에 반영해서 생긴다.
- 정지 상태에서도 화살표는 유지해야 한다.
- 작은 각도 변화는 무시하고, 큰 변화도 부드럽게 보간하면 UX가 좋아진다.

## 접근
1. 최근 적용 방향값과 마지막 업데이트 시간을 저장한다.
   - verify: 전역 상태가 추가된다.
2. 각도 차이를 0~180도 기준으로 계산하는 헬퍼를 추가한다.
   - verify: 359도↔1도 같은 경계값도 짧은 방향으로 계산한다.
3. 작은 변화는 무시하고, 일정 주기보다 빠른 업데이트도 건너뛴다.
   - verify: 6도 미만 변화와 120ms 이내 이벤트는 회전을 갱신하지 않는다.
4. 큰 변화는 원형 보간으로 부드럽게 반영한다.
   - verify: SVG transform에 smoothing된 heading을 사용한다.
5. 문법 검증 후 커밋/푸시한다.
   - verify: `node --check script.js` 통과.
